Step 1 · Installation

Install Analytics v5 + CavAi Brain

CavBot starts with two scripts. No build step, no SDK ceremony. Place both in your layout, template, or HTML shell so they load across the routes you want under guard.



Tracker snippet panel

Required attributes: data-project-key and data-site. Add the CavAi Brain script after Analytics v5. Optional: data-badge="true" to mount the badge.


Analytics + Brain snippet
<script
  src="https://cdn.cavbot.io/sdk/v5/cavai-analytics-v5.js"
  data-project-key="YOUR_PROJECT_KEY"
  data-site="YOUR_SITE_ID"
  data-badge="true"
  defer
></script>
<script
  src="https://cdn.cavbot.io/sdk/brain/v1/cavai.min.js"
  defer
></script>

  • Placement: in <head> with defer (recommended) or before </body>.
  • IDs: anonymousId is project-scoped; sessionKey groups activity windows.
  • Defaults: no emails, names, or account identifiers required to operate.

Step 4 · Privacy & retention

Production-safe defaults

CavBot is built for operational signal. IDs are anonymous and project-scoped. Payloads are intentionally small. Retention can be configured per project to match your data posture.



IDs & identifiers


  • Project-scoped anonymousId (UUID) per visitor.
  • sessionKey groups events by activity window.
  • No names, emails, or account identifiers required by default.

IP & location


  • Optional IP handling; when enabled, truncation occurs before storage.
  • Coarse geography only for routing and anomaly detection.
  • No long-term storage of raw IPs as a default posture.

Retention controls


  • Raw events retained for an operational window (example: 90 days).
  • Aggregates can be retained longer for trends.
  • Project toggles can disable event types or performance sampling.

Install path

Add the 404 surface and emit typed events

This is the minimal installation process. The goal: your 404 becomes a recovery surface — and CavBot receives clean, typed telemetry.

Step 1 · Ensure the tracker is present Required

The 404 surface assumes the CavBot tracker exists on the site (global layout / template). 

<script
  src="https://cdn.cavbot.io/sdk/v5/cavai-analytics-v5.js"
  data-project-key="YOUR_PROJECT_KEY"
  data-site="YOUR_SITE_ID"
  data-badge="true"
  defer
></script>

Step 2 · Route your 404 to the arcade recovery surface Surface

Render the arcade recovery experience on your 404 route (framework or static 404). Emit cavbot_404_view on entry. 

// Example: Next.js (pages router)
// pages/404.js
import CavBotArcadeRecovery from "../components/CavBotControlRoom";

export default function NotFoundPage() {
  if (typeof window !== "undefined" && window.cavbotAnalytics) {
    window.cavbotAnalytics.track("cavbot_404_view", {
      route: window.location.pathname,
      pageType: "404-arcade-recovery"
    });
  }

  return (
    <CavBotArcadeRecovery difficulty="standard" theme="grid-lab" />
  );
}

When the visitor plays / interacts, emit clean, typed outcomes. Keep payloads small.

cavbot_catch cavbot_miss cavbot_idle

Step 2 · Set keys Required

Configure project key and site id

data-project-key maps to your project. data-site keeps production and staging cleanly separated.

  • production — live traffic
  • staging — pre-release validation
  • preview — ephemeral builds

IDs are anonymous and project-scoped. CavBot is designed for route-level operational insight.

Step 3 · Enable the badge Optional

Mount the on-site badge

Enable the badge via snippet attribute (data-badge="true") or through an explicit init call. 

window.cavbotAnalytics.init({
  projectKey: "YOUR_PROJECT_KEY",
  environment: "production",
  cavbotBadge: true
});

Badge interactions can be tracked (impression, hover, click) without collecting personal identifiers.

Step 4 · Wire your 404 surface Signal

Convert 404s into structured events

Route your 404 experience through the arcade recovery component and emit typed events. Example: Next.js. 

// pages/404.js
import CavBotArcadeRecovery from "../components/CavBotControlRoom";

export default function NotFoundPage() {
  if (typeof window !== "undefined" && window.cavbotAnalytics) {
    window.cavbotAnalytics.track("cavbot_404_view", {
      route: window.location.pathname,
      pageType: "404-arcade-recovery"
    });
  }

  return (
    <CavBotArcadeRecovery difficulty="standard" theme="grid-lab" />
  );
}
  • cavbot_404_view — entry event
  • cavbot_catch, cavbot_miss, cavbot_idle — interaction signal
